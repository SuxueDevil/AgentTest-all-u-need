# CLAUDE.md

## 项目概述

基于 Vite + Vue 3 + TypeScript 的前端项目，使用 Tailwind CSS 作为样式框架。

## 每日计划维护（重要）

每次修改代码时，必须同步更新 `doc/plan/` 下的当日计划文件。

### 文件命名

- 格式：`doc/plan/M.DD-plan.md`（例如 `5.15-plan.md`）
- 每天一个文件，旧文件保留作为历史记录
- 当天首次操作时自动创建该文件

### 计划文件结构

每个 plan 文件必须包含以下板块：

```markdown
# M.DD 开发计划

## 当前进度概览
<!-- 简要说明今天做了什么、整体进展如何 -->

## 下一步待做功能
<!-- 列出接下来要开发的功能，每项用 - [ ] 标记 -->

## API / 接口清单
<!-- 今天涉及或计划涉及的接口，包括路径、方法、用途 -->

## 已完成事项
<!-- 今天已完成的功能/修复，每项用 - [x] 标记 -->

## 阻塞 / 风险
<!-- 当前遇到的问题、阻塞项、潜在风险 -->
```

### 更新规则

1. **每次代码修改后**，检查当日 plan 文件是否存在，不存在则创建
2. 根据本次改动，更新对应板块：
   - 完成了某个功能 → 移到「已完成事项」，勾选 `- [x]`
   - 新增了接口 → 补充到「API / 接口清单」
   - 发现了新需求 → 加到「下一步待做功能」
   - 遇到了问题 → 记录到「阻塞 / 风险」
3. 「当前进度概览」保持简洁，一两句话即可
4. 不要为了更新而更新——有实质性变化才改
